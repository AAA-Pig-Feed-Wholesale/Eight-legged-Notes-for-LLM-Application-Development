# LLM八股

好，我把前面那两大坨 **LLM 八股 + Transformer 八股** 都给你整理成「可以在面试里直接说」的口语版本。

你不用每句都死背，记住结构+关键词就行。

------

## 一、LLM 常见八股（口语版）

### 1. 什么是大型语言模型？和传统 NLP 模型有什么区别？

> 我理解的大模型，还是一个基于 Transformer 的概率语言模型，本质上就是“给前文，预测下一个 token 的分布”。
>
> 跟传统 NLP 不同的是三个点：数据量更大、参数规模更大；不用为每个任务单独设计结构，只要换指令就能做各种事；而且通过预训练 + 指令微调 + 对齐，一套模型能覆盖聊天、代码、工具调用等很多场景。

------

### 2. 简单讲一下 Transformer 和 Self-Attention

> Transformer 的核心就是 Self-Attention。每个词会算出 Q、K、V，然后用 Q 去和所有 K 做相似度，算出一组权重，再对 V 做加权和。
>
> 好处是整段序列可以并行算，而且任何两个位置之间的关系都能直接建模，比 RNN 那种一步一步传的信息更灵活。

------

### 3. BERT 和 GPT / Qwen / LLaMA 的结构差别？

> BERT 是 **encoder-only** 的双向结构，预训练是做 MLM，更适合分类、理解任务。
>
> GPT、Qwen、LLaMA 这类是 **decoder-only** 自回归结构，只看左边，预训练就是预测下一个 token，更适合做生成。
>
> 所以现在对话式大模型基本都是 GPT 这条路，BERT 常用来做 encoder 或 embedding。

------

### 4. 位置编码是干嘛的？绝对位置 vs 相对 / RoPE？

> 注意力本身不带顺序感，所以要加位置编码告诉模型“谁在前谁在后”。
>
> 绝对位置就是每个位置加一个固定向量；相对位置或 RoPE 是让模型更关心 token 之间的距离。
>
> 新一代大模型，比如 Qwen、LLaMA，基本都是 RoPE 这类相对位置编码，对长上下文和位置外推更友好。

------

### 5. LLM 的训练一般分哪几阶段？

> 通常是三段：预训练、SFT、对齐。
>
> 预训练在超大语料上学语言和常识；SFT 用「指令–回复」让模型学会按人类说话方式回答；
>
> 最后 RLHF / DPO 这类对齐方法，用偏好数据约束模型：该拒绝的拒绝、该保守的保守。

------

### 6. 什么是 SFT？优点和局限？

> SFT 就是监督微调，用高质量「指令–理想回答」继续训模型。
>
> 它的优点是特别简单、特别稳，适合调风格、调格式、调工具调用。
>
> 局限是它只能学“平均的好回答”，但没法表达“这个回答明显比另一个好多少”，所以偏好排序、安全性一般还要叠 DPO 或 PPO。

------

### 7. RLHF / PPO / DPO 大概是啥思路？

> RLHF 典型做法是：先用 SFT 得到模型，然后用人类偏好数据训一个奖励模型，再用 PPO 让策略模型在这个奖励下得到更高收益。
>
> DPO 则更简单，直接拿好答/坏答成对数据，把语言模型当策略，优化一个对比损失，让模型更偏向好答。
>
> 总结一下：PPO 是 RL 套路，DPO 是“直接用偏好数据优化 LM”的套路，工程上 DPO 更轻量。

------

### 8. 全量微调 vs LoRA 这种 PEFT，有什么差异？

> 全量微调就是所有参数都跟着更新，灵活但特别吃显存、存储。
>
> LoRA 这种 PEFT，只在某些层加低秩矩阵，训练的时候只调这小部分参数，推理时跟原权重相加就行。
>
> 资源紧、任务多的时候，我会优先选 LoRA；像 Kaggle 那个项目，我就同时做过 DeBERTa-small 全参和 DeBERTa-large+LoRA 的对比。

------

### 9. 生成时常见的采样策略：温度 / Top-k / Top-p？

> 模型输出的是概率分布，怎么从分布里抽样就决定了生成风格。
>
> 温度是调“保守程度”，温度低一点更稳、低多样；Top-k 是只在前 k 个候选里采样；Top-p 是取累计概率到 p 的最小集合再采样。
>
> 实际用的时候一般会组合，比如 T=0.7，加个 top-p，既不太死板也不瞎飞。

------

### 10. KV Cache 是干嘛的？为什么能加速？

> 自回归生成时，如果每个 token 都重算整段序列，复杂度是 O(n²)，很慢。
>
> KV Cache 的思路是：把前面所有 token 的 K/V 存起来，后面每一步只需要算新 token 的 Q，然后和缓存的 K/V 做 attention。
>
> 这样每一步是增量计算，长文本生成速度能快很多。

------

### 11. RAG 和微调各自适合什么场景？为什么很多先上 RAG？

> 微调是直接改模型参数，让模型记住某类行为；RAG 是把知识放在外部库里，模型负责理解和组织语言。
>
> 知识更新频繁、需要溯源的场景，比如医疗、法律、金融，我更倾向先用 RAG，因为只要更新索引，不用重训模型，还能说出“这句话是根据哪几段文档来的”。
>
> 像偏好预测、分类这种结构化任务，就更适合用我 Kaggle 那套微调流程。

------

### 12. 文本 Embedding 在 RAG 里扮演什么角色？怎么选模型？

> Embedding 就是把文本变成向量，让语义相近的内容在空间里更近。
>
> 在 RAG 里，它直接决定语义检索好不好用：query 和文档块都被 embed 到同一个空间，再用相似度做召回。
>
> 选模型要看语种和领域，我的医疗项目里就是从 all-MiniLM 做 baseline，再用 ragas 和人工抽检发现中医相关表现一般，然后换成 Qwen-3 text-embedding-v3，明显提升了中文医疗语料的召回。

------

### 13. 幻觉为什么会出现？你一般怎么抑制？

> 幻觉本质上就是模型“被训练成必须说点什么”，即使没证据也要编。
>
> 预训练目标从来没人教它“可以说不知道”，而且数据里也有不少噪声。
>
> 我的做法就是三层：提高检索质量；在 prompt 里明确要求“只能基于证据，有疑问必须说不确定＋建议线下就诊”；对高风险内容加规则过滤，该拒答就拒答。

------

### 14. 常见评估指标有哪些？你怎么用？

> 训练过程常看 loss / perplexity，属于模型层面的。
>
> 文本生成会有 BLEU、ROUGE，这些在翻译、摘要比较传统。
>
> RAG 的话，我会看 ragas 里的 context_recall 和 context_precision，再配合少量人工打分；真正落地时，还要看业务侧指标，比如用户满意度、问题解决率。

------

### 15. Kaggle 偏好预测为什么用 Log Loss？你怎么理解？

> Log Loss 本质就是交叉熵，它关心的不只是分类对不对，还关心概率分布是不是“有自知之明”。
>
> 高置信错误会被惩罚得特别重，所以能鼓励模型在不确定的时候不要给 0.99。
>
> 在 Kaggle 项目里，我就是一边看 Log Loss，一边看 Macro-F1 和混淆矩阵，两边一起看才能知道“既不过拟合优势类，也照顾到弱势类”。

------

### 16. 什么是 tokenizer？为什么大模型里“token”这么重要？

> tokenizer 本质上是把原始文本切成模型能处理的最小离散单元，也就是 token。
>
> 大模型不是直接看“字”或“词”，而是先把文本编码成 token id，再做 embedding 和后续计算。
>
> token 之所以重要，是因为：
>
> - 上下文长度按 token 算，不按字符算；
> - 计费通常按 token 算；
> - 检索拼接、Prompt 设计、缓存命中，本质都和 token 分布有关。
>
> 所以工程里我会特别注意“同样一句话在不同 tokenizer 下 token 数差很多”这件事，尤其是中英混合和代码场景。

------

### 17. MoE（Mixture of Experts）模型是什么？和稠密模型有什么区别？

> 稠密模型是每一层参数都参与计算；MoE 则是在某些层里只激活少数几个 expert，让模型总参数很大，但每次前向真正参与计算的参数没那么多。
>
> 它的优势是：
>
> - 总参数规模可以更大；
> - 计算成本不一定线性上升；
> - 在相同激活参数预算下，效果往往更好。
>
> 但它也更难训，典型难点包括：
>
> - expert 负载不均衡；
> - 路由不稳定；
> - 分布式训练和推理更复杂。
>
> 面试里一句话可以概括为：MoE 是用更复杂的路由机制，换更高的参数利用率。

------

### 18. 大模型推理为什么常做量化？4-bit / 8-bit 的意义是什么？

> 因为大模型推理的核心瓶颈之一是显存和带宽。量化就是把原本 16-bit 或更高精度的权重压缩到 8-bit、4-bit，减少显存占用和访存成本。
>
> 它的价值主要有两个：
>
> - 让原本放不下的模型能部署；
> - 在很多场景下降低推理成本。
>
> 代价是：
>
> - 可能有精度损失；
> - 某些任务，尤其是推理和长链条任务，对量化更敏感；
> - 工程兼容性也会更复杂。
>
> 所以量化的本质是“在成本和效果之间做权衡”，不是无脑越低越好。

------

### 19. 什么是 speculative decoding / continuous batching 这类推理优化？

> 这类技术本质上都在解决同一个问题：大模型生成慢、吞吐低。
>
> - speculative decoding：先让一个小模型草拟几个 token，大模型再验证，验证通过就一次接受多 token，从而提速；
> - continuous batching：不是等一整批请求一起处理，而是让新请求可以动态插队进 batch，提高 GPU 利用率。
>
> 面试里不用讲太深公式，讲清楚目标就行：
>
> > 前者是在减少“大模型逐 token 生成”的低效，后者是在提升服务层的吞吐。

------

### 20. 长上下文是不是会取代 RAG？

> 我觉得不会完全取代，二者解决的问题不一样。
>
> 长上下文的好处是能直接塞更多材料，减少切分和检索误差；
>
> 但 RAG 仍然有几个不可替代的价值：
>
> - 知识可更新；
> - 可溯源；
> - 不需要每次都把海量材料完整塞进上下文；
> - 在超大知识库里成本通常更可控。
>
> 所以现实里更常见的是“长上下文 + RAG”结合，而不是二选一。

------

## 二、Transformer 常见八股（口语版）

### 1. Transformer 是什么？为什么可以替代 RNN/LSTM？

> Transformer 就是“Attention is All You Need”那篇里的结构，用自注意力取代循环。
>
> 关键是它可以并行算整段序列，不像 RNN 那样必须一步一步来，而且任意两个位置都能直接建立联系，对长距离依赖特别友好。
>
> 所以一上来它就把机器翻译干翻了 RNN 系列，现在大模型基本都是基于它。

------

### 2. 一个标准的 Transformer Encoder Block 有哪些部分？

> 一个 Encoder Block 就两个大块：多头自注意力 + 前馈网络。
>
> 外面统一包一层残差连接，再加 LayerNorm。
>
> 有的实现是“先子层再 Norm”，有的是 Pre-LN，现在大模型一般用 Pre-LN，训练更稳。

------

### 3. Self-Attention 里 Q、K、V 怎么算？什么意思？

> 输入一段序列，每个位置先通过三个线性层，分别得到 Q、K、V。
>
> 用 Q 和所有 K 做点积，缩放一下、过 softmax，就得到一组注意力权重，再对 V 做加权求和就是输出。
>
> 简单理解：Q 是“我要查什么”，K 是“我是什么标签”，V 是“我身上的信息”。
>
> 在 Transformer 的自注意力机制中，**K（Key）、V（Value）、Q（Query）** 是三个核心向量，它们共同决定了模型如何捕捉序列中不同位置之间的依赖关系。下面我用口语化的方式解释它们的定义和作用，并结合原始公式说明。
>
> ### 1. **K、V、Q 的基本定义**
>
> - **Q（Query）**：代表“查询向量”，相当于当前 token 想要询问的问题。比如在生成“猫坐在毛毯上，因为它很暖和”时，“它”的 Q 向量会查询前文哪个词与“它”最相关（如“毛毯”）。
>
> - **K（Key）**：代表“键向量”，相当于每个 token 的“身份标签”。其他 token 的 Q 会通过计算与 K 的相似度来筛选需要关注的 token。
>
> - **V（Value）**：代表“值向量”，存储每个 token 的实际内容信息。注意力权重计算完成后，会对 V 进行加权求和，生成包含上下文信息的新向量。
>
> 简单类比：**Q 像搜索关键词，K 像文档的标题标签，V 像文档的具体内容**。模型通过 Q 和 K 的匹配程度，决定从 V 中提取多少信息。
>
> ### 2. **K、V、Q 的生成方式**
>
> 在公式中，K、V、Q 是通过输入序列的线性变换得到的：
>
> - 每个 token 的嵌入向量 x 会分别与三个权重矩阵 Wq、Wk、Wv 相乘： 
>
> - q=xWq（生成 Q）
>
> - k=xWk（生成 K）
>
> - v=xWv（生成 V） 这些权重矩阵是模型在训练过程中学习到的参数。例如，如果输入序列有 6 个 token，每个 token 的嵌入维度是 3，那么 Q、K、V 的维度可能是 2 或 4，具体取决于投影矩阵的设定。
>
> ### 3. **在原始公式中的作用**
>
> 自注意力的核心公式如下：
>
> Attention(Q,K,V)=softmax(dkQKT)V
>
> - QKT：Q 和 K 的点积计算相似度，得到注意力分数。分数越高，表示两个 token 相关性越强。
>
> - **缩放** dk：防止点积结果过大导致 softmax 梯度消失。
>
> - **softmax**：将分数转化为概率分布（注意力权重），确保权重总和为 1。
>
> - **加权求和**：最后用注意力权重对 V 进行加权，得到输出向量。例如，如果“它”与“毛毯”的注意力权重为 0.9，那么输出向量会包含 90% 的“毛毯”信息。
>
> ### 4. **为什么需要三者分工？**
>
> - **分工优势**：如果只用 Q 和 K，模型只能知道哪些 token 相关，但无法提取具体信息；引入 V 后，模型能动态融合不同 token 的内容。这种设计让 Transformer 可以并行处理序列，并直接建模长距离依赖，优于 RNN 的逐步传递机制。
>
> - **多头注意力扩展**：通过多个“头”并行计算不同的 Q、K、V，模型可以同时关注语法、语义等多方面信息，增强表达能力。
>
> ### 总结
>
> K、V、Q 是自注意力机制的“发动机”：**Q 负责提问，K 负责匹配，V 负责提供答案内容**。这种设计使得 Transformer 能够灵活捕捉全局上下文，成为现代大模型的基石。如果你对照代码实现（比如 HuggingFace 的 Transformer 库），会发现这些向量直接对应着模型中的 `query`、`key`、`value` 线性层。

------

### 4. 为什么注意力分数要除以 (\sqrt{d_k})？

> 如果不除，随着向量维度变大，点积的数值会越来越大，softmax 之后变得特别尖，梯度就会非常小。
>
> 除以 (\sqrt{d_k}) 就是做一个归一化，让分布保持在一个比较舒适的范围里，训练比较稳定。

------

### 5. 为什么要多头注意力，而不是单头？

> 单头注意力就像一个视角在看句子，多头就是多个视角同时看。
>
> 不同头可以学不同的关系，比如一个头学短距离，另一个头学长距离，有的头专门看实体、有的看句法。
>
> 最后把各个头拼起来，表达能力更强。

------

### 6. Self-Attention 和 Cross-Attention 有啥不同？

> Self-Attention 是 Q/K/V 都来自同一个序列，用来看内部谁和谁相关。
>
> Cross-Attention 是 Q 来自当前序列，K/V 来自另一个序列，比如翻译时 Q 是译文的 hidden state，K/V 是源文编码结果。
>
> Encoder 里基本是 Self-Attn，Encoder-Decoder 结构的 Decoder 里会有 Self-Attn + Cross-Attn。

------

### 7. Masked Self-Attention 是什么？为什么要 Mask？

> 在自回归生成里，第 t 个 token 不能看到未来，只能看 1 到 t-1。
>
> 所以在 Decoder 里会加一个下三角 Mask，把看未来的注意力权重直接屏蔽掉。
>
> 这样就能保证生成是因果的，不会“提前偷看答案”。

------

### 8. Self-Attention 的复杂度为什么是 O(n²)？带来什么问题？

> 对长度为 n 的序列，要算一个 n×n 的注意力矩阵，每对 token 都算相关性，所以时间和空间都是 O(n²)。
>
> n 小还好，一旦文本稍长，比如几万 token，显存和时间都顶不住。
>
> 这也是大家做各种 Longformer、Performer、FlashAttention 的原因，就是想把这个 n² 的成本降下来。

------

### 9. 有哪些常见的“长上下文 / 注意力加速”的思路？

> 大概有几类：
>
> 一类是稀疏注意力，比如 Longformer，只在局部窗口或少量全局 token 上算；
>
> 一类是做核函数近似，比如 Performer，用可分解的核近似来把复杂度降到近似线性；
>
> 再就是工程上的优化，比如 FlashAttention，从内存访问和算子层面提升实际吞吐。
>
> 实际用的时候一般不会自己造轮子，会选一个库里支持得比较好的方案。

------

### 10. 为什么要 Residual + LayerNorm？Pre-LN 和 Post-LN 有啥区别？

> 残差是让梯度和信息可以“绕过”中间层，防止太深的网络训练不动，也让每层只学增量。
>
> LayerNorm 是对每个样本在特征维度上做归一化，让激活分布更平稳。
>
> Post-LN 是子层之后再 Norm，Pre-LN 是先 Norm 再进子层，现在大部分大模型用 Pre-LN，因为深层训练更稳定。

------

### 11. BERT 和 DeBERTa 在 Transformer 上有哪些改动？

> BERT 基本就是标准 Encoder 堆叠，配上一些预训练头。
>
> DeBERTa 的特点是所谓「Disentangled Attention」，把内容和位置信息拆开建模，再用相对位置编码，提升了对长依赖和位置泛化的能力。
>
> 所以在很多下游任务上，DeBERTa 同规模下会比 BERT/RoBERTa 强一些，我 Kaggle 项目就是用的 DeBERTa-v3。

------

### 12. Decoder-only LLM 和 Encoder-Decoder Transformer 的区别？为什么大模型更偏向 Decoder-only？

> Encoder-Decoder 是“先编码输入，再用 Decoder 在编码结果上自回归生成”，这种结构一开始就是为翻译设计的。
>
> Decoder-only 是把所有东西串成一个序列，在同一个堆栈里自回归生成。
>
> 大模型更偏向 Decoder-only，一方面是预训练目标简单统一，另一方面任何任务都可以改写成「指令 + 输入 → 输出」的形式，统一范式、工程实现更简单。
