# 大模型应用开发八股总纲

------

## 1. 这套八股该怎么用？

这个文件夹现在可以按下面 8 个模块来记：

1. `LLM八股.md`：模型基础、Transformer、训练阶段、采样、评估
2. `prompt工程八股.md`：Prompt 设计、调试、A/B、约束输出
3. `RAG八股.md`：知识库、检索、重排、评估、线上优化
4. `大模型微调八股.md`：SFT、LoRA、QLoRA、对齐、评估、训练工程
5. `Agent八股.md`：规划、工具调用、记忆、评估、Agent 风险
6. `python八股.md`：语言基础、异步并发、日志、异常、工程实现
7. `后端八股.md`：传统后端基础 + LLM后端扩展，覆盖接口、缓存、队列、数据库、可观测、稳定性
8. `前沿Agent生态八股.md`：MCP、Skills、OpenClaw、A2A、Computer Use 等前沿 Agent 生态

如果你是为了面试记忆，不建议按文件顺序死背，而建议按“从底到上”的链路去记。

------

## 2. 一条主线：从模型能力到系统落地

你可以把整套知识压缩成一句话：

> 大模型应用开发，本质上是在做“模型能力 + 外部知识 + 工程系统 + 业务约束”的组合优化。

进一步展开就是：

1. 模型先天能力来自预训练
2. Prompt 决定短期调用效果
3. RAG 负责接入外部知识
4. 微调负责改变模型行为和任务适配
5. Agent 负责多步决策与工具编排
6. Python 和后端负责把这些能力变成稳定产品
7. 前沿协议和平台决定你能不能讲清楚“下一代 Agent 生态怎么搭”

面试里，你只要能顺着这条主线讲，层次通常不会差。

## 4. 面试最常考的 12 个主问题

这 12 个问题建议优先背，因为它们基本能覆盖 80% 的面试：

1. 什么是大语言模型？和传统 NLP 有什么区别？
2. Prompt、RAG、微调分别解决什么问题？怎么选？
3. 典型 RAG 系统怎么设计？有哪些坑？
4. 什么情况下要微调？为什么很多业务先上 RAG？
5. SFT、LoRA、QLoRA 的区别是什么？
6. DPO、RLHF、GRPO 分别是什么，怎么选？
7. Agent 和普通工作流有什么区别？
8. 怎么评估一个大模型系统好不好？
9. 如何降低幻觉？
10. 如何做一个生产可用的 LLM 服务？
11. Python 在大模型应用开发里最重要的能力点有哪些？
12. 结合项目，讲一个你做过的完整优化闭环。

## 5. 面试回答的标准结构

不管问哪个方向，尽量都按这个结构答：

1. 先给定义
2. 再讲核心原理
3. 再讲工程权衡
4. 最后落到实际场景

比如回答“RAG 和微调怎么选”时，可以这样组织：

> 先说定义：RAG 是把知识放外部，微调是改模型参数。  
> 再说原理：RAG 强在知识更新和可溯源，微调强在行为固化和格式对齐。  
> 再说 trade-off：RAG 系统复杂但灵活，微调训练贵但推理简洁。  
> 最后落地：知识更新快的客服、医疗、法律问答优先 RAG；强结构化输出、分类打分、工具调用协议更适合微调。

## 7. 一套最适合记忆的口诀

可以记成这 7 句：

- 基础能力看 `LLM`
- 临时调优看 `Prompt`
- 外部知识看 `RAG`
- 行为固化看 `微调`
- 多步决策看 `Agent`
- 代码实现看 `Python`
- 线上落地看 `后端`

再压缩成一句：

> 会解释模型，会设计知识链路，会做工程落地，会讲 trade-off，这就是大模型应用开发面试的主干。

## 9. 最后应该形成什么能力？

理想状态不是“背会 200 个问答”，而是形成下面这 4 个输出能力：

1. 能解释概念，不空泛
2. 能比较方案，有取舍
3. 能结合项目，不虚
4. 能说清链路，不碎片

如果你能做到这四点，面试里就已经不是“背八股”，而是在展示系统性。

------

## 10. 终极收尾话术

如果面试最后让你概括自己对大模型应用开发的理解，你可以这样说：

> 我理解大模型应用开发不是只会调模型 API，而是要把模型能力、Prompt、RAG、微调、Agent、后端工程和评测体系串起来。不同业务问题的关键不是盲目上最复杂方案，而是先判断问题到底是知识问题、行为问题，还是系统问题，再选 Prompt、RAG、微调或 Agent 这些手段去组合解决。
